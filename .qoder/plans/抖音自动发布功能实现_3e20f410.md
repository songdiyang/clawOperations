# 抖音开放平台自动发布功能实现计划

## 背景

项目 ClawOperations 当前为空白状态（仅有 README.md），需从零构建。目标是通过抖音开放平台（https://developer.douyin.com/）官方 API 实现程序化视频自动发布功能，供 ClawBot 运营机器人调用。

## 技术选型

- **运行时**: Node.js 18+
- **语言**: TypeScript（严格模式）
- **HTTP 客户端**: axios
- **日志**: winston
- **定时任务**: node-cron
- **配置管理**: dotenv
- **测试框架**: jest

## Task 1: 项目脚手架初始化

- `config/default.ts`: 默认配置常量
- `src/utils/logger.ts`: winston 日志封装
- `src/utils/retry.ts`: 指数退避重试工具函数

## Task 2: 类型定义与数据模型

**目标**: 定义所有 API 交互相关的 TypeScript 类型。

**具体内容** (`src/models/types.ts`):
- **认证相关**: `OAuthConfig`, `TokenResponse`, `TokenInfo`
- **视频上传相关**: `UploadConfig`, `ChunkUploadInitResponse`, `ChunkUploadPartResponse`, `UploadCompleteResponse`
- **视频发布相关**: `VideoPublishOptions`（标题、描述、hashtags、@mentions、POI、小程序、商品链接、定时发布时间）, `VideoCreateResponse`
- **通用类型**: `DouyinApiResponse<T>`, `DouyinApiError`, `RetryConfig`
- **发布服务相关**: `PublishTaskConfig`（完整的发布任务配置，包含视频路径 + 发布参数）

**无依赖，可与 Task 1 并行。**

## Task 3: 抖音 API 基础客户端

**目标**: 实现与抖音 API 通信的基础层。

**具体内容** (`src/api/douyin-client.ts`):

- 日志记录所有 API 请求/响应

**依赖 Task 1、Task 2。**

## Task 4: OAuth2.0 认证模块

**目标**: 实现抖音 OAuth2.0 完整认证流程。

**具体内容** (`src/api/auth.ts`):
- `getAuthorizationUrl()`: 生成授权页面 URL（scope: `video.create`, `video.upload` 等）
- `getAccessToken(authCode)`: 用授权码换取 access_token + refresh_token
- `refreshAccessToken(refreshToken)`: 刷新过期 token
- `isTokenValid()`: 检查 token 是否过期
- Token 自动续期机制（在 API 调用前自动检查并刷新）

**抖音 OAuth2.0 端点**:
- 授权页: `https://open.douyin.com/platform/oauth/connect/`
- 获取 token: `/oauth/access_token/`
- 刷新 token: `/oauth/renew_refresh_token/`

**依赖 Task 3。**

---

## Task 5: 视频上传模块

**目标**: 实现视频文件上传到抖音服务器，支持分片上传（大文件）和完整上传。

**具体内容** (`src/api/video-upload.ts`):
- `uploadVideo(filePath)`: 自动根据文件大小选择上传方式
  - 小于 128MB：完整上传 `POST /api/v2/video/upload/`
  - 大于 128MB：分片上传流程
- **分片上传流程**:
  1. `initChunkUpload()`: 初始化分片上传，获取 upload_id
  2. `uploadChunk(uploadId, chunkData, partNumber)`: 上传单个分片
  3. `completeChunkUpload(uploadId)`: 合并分片，完成上传
- 分片大小可配置（默认 5MB）
- 上传进度回调支持
- 断点续传支持（记录已上传的分片）
- 文件类型/大小校验（`src/utils/validator.ts`）

**依赖 Task 3、Task 4。**

---

## Task 6: 视频发布模块

**目标**: 实现视频内容创建和发布，支持全部内容配置能力。

**具体内容** (`src/api/video-publish.ts`):
- `createVideo(options: VideoPublishOptions)`: 调用 `/api/v2/video/create/`
- **支持的发布参数**:
  - `title`: 视频标题
  - `description`: 视频描述文案
  - `hashtags`: 话题标签数组（自动拼接 # 前缀）
  - `atUsers`: @提及用户列表（open_id 数组）
  - `poiId`: 地理位置 POI ID
  - `poiName`: 地理位置名称
  - `microAppId`: 小程序 ID（商业挂载）
  - `microAppTitle`: 小程序标题
  - `microAppUrl`: 小程序链接
  - `articleId`: 商品 ID（商业链接）
  - `schedulePublishTime`: 定时发布时间（Unix 时间戳）
- 参数校验：标题长度、描述长度、hashtag 数量限制等
- 发布结果返回：video_id、分享链接等

**依赖 Task 3、Task 4。可与 Task 5 并行。**

## Task 7: 发布业务编排服务

**目标**: 将上传和发布流程串联为完整的业务流程。

**具体内容** (`src/services/publish-service.ts`):
- `publishVideo(config: PublishTaskConfig)`: 一站式发布
  1. 校验视频文件和发布参数
  2. 上传视频到抖音服务器
  3. 获取 video_id
  4. 调用发布接口创建视频
  5. 返回发布结果
- 支持从本地文件路径或远程 URL 发布
- 错误处理与回滚（上传失败不执行发布）
- 发布状态跟踪

**具体内容** (`src/services/scheduler-service.ts`):
- `schedulePublish(config, publishTime)`: 注册定时发布任务
- `cancelSchedule(taskId)`: 取消定时任务
- `listScheduledTasks()`: 列出待发布任务
- 基于 node-cron 的本地定时调度

**依赖 Task 5、Task 6。**
